Step-by-Step Pipeline

Each stage of the Streamlit dashboard explained in plain language, so any audience can follow the logic from raw data to final model.

1

Data Upload & Exploratory Analysis

Users upload a CSV file containing symptom/feature columns and a disease label. The dashboard instantly renders a data profile: shape, dtypes, null counts, and a correlation heatmap — making it easy to spot quality issues before training.

Pandas Profiling Correlation Heatmap Class Distribution
2

Preprocessing: Missing Values & Outliers

Null values are handled with user-selected strategies (mean, median, mode, or drop). Outliers are detected via IQR and optionally clipped. All transformations are logged so the user can audit every step.

IQR Clipping Mean / Median Imputation Audit Log
3

Encoding & Feature Scaling

Categorical symptom columns are one-hot or label-encoded. Numeric features are normalised with Standard or MinMax scaling. The encoded dataset preview and a feature importance bar chart are shown live.

One-Hot Encoding StandardScaler Feature Importance
4

Model Training & Cross-Validation

Select from five classifiers — Logistic Regression, Random Forest, SVM, KNN, and MLP Neural Network. Training is performed with stratified k-fold cross-validation. Live progress bars update as each fold completes.

Random Forest MLP Neural Net SVM K-Fold CV
5

Evaluation Metrics & Confusion Matrix

Results are displayed as accuracy, precision, recall, and F1-score tables alongside interactive confusion matrices. A side-by-side model comparison chart helps identify the best performer at a glance.

Confusion Matrix F1 Score Model Comparison
6

Hyperparameter Tuning

GridSearchCV and RandomizedSearchCV are available. Users define search grids via sliders and the app runs the search, displays the best parameters, and retrains the final model — all inside the dashboard.

GridSearchCV RandomizedSearch Best Params
7

Prediction & Export

Enter new symptom values manually or upload a batch CSV. The tuned model outputs predicted diseases with confidence percentages. Results can be downloaded as a labelled CSV for clinical or academic use.

Live Inference Confidence % CSV Export

Core Technologies & Architecture

A look under the hood at the robust frameworks and libraries powering this end-to-end machine learning pipeline.

The system is built entirely on industry-standard Python data science libraries:

🎨
Streamlit & Plotly

Provides a highly reactive, component-based dashboard UI with rich, interactive data visualisations.

🐼
Pandas & Profiling

Handles core tabular data manipulation, advanced cleaning, and automated exploratory data analysis reports.

🤖
Scikit-Learn

Powers the comprehensive ML backend, including diverse classifiers, scalers, and rigorous cross-validation.

⚡
NumPy & SciPy

Underpins the heavy mathematical operations, matrix transformations, and dynamic hyperparameter search spaces.

1. Ingestion Dynamic CSV schema validation
2. Preprocess Stateful imputation & feature scaling
3. Train Parallelised Stratified K-Fold CV
4. Optimise Automated Grid & Random searches
5. Inference Real-time prediction confidence scoring
6. Export Persistent model state & batch results

Key Takeaways

What makes this project stand out and what the viewer should walk away understanding.

🔬

Interpretable ML

Every prediction comes with feature importance scores and SHAP-ready outputs — making the model's reasoning transparent, not a black box.

⚡

No-Code Pipeline

The entire workflow — from upload to download — runs inside the Streamlit UI. No coding skills needed by the end user to derive predictions.

📊

Real-Time Visuals

Interactive Plotly charts update live as models train. Confusion matrices and accuracy curves give instant visual feedback on model quality.

🧪

Rigorous Validation

Stratified k-fold cross-validation ensures the accuracy isn't inflated by lucky splits, giving reliable estimates on imbalanced disease classes.

🎛️

Tuning Built-In

Grid and randomised search run directly in the app. Sliders let the user define search ranges without touching config files or Python code.

🌐

Future-Ready

Architecture is modular — ready to be wrapped in a FastAPI backend, containerised with Docker, and extended with deep learning or LLM-based symptom parsers.
